Let hosts type ticket `metadata`, two ways. Pass any Standard Schema as `config.metadataSchema` and `createSpindesk` infers the metadata type from it — no type argument — while also validating `metadata` at runtime (`400` on failure). A configured schema is treated as a guarantee: metadata is required on create unless the schema accepts an empty value (`{}`, `null`, or `undefined`); updates validate only when metadata is included. Or pass a type argument, `createSpindesk<MyMeta>(…)`, for compile-time typing only. Either way `metadata` flows end-to-end (create/update bodies, ticket responses, and the typed client); `createSpindeskClient<MyMeta>()(…)` pins the same shape on the client (now called in two steps, since TypeScript can't infer the client options while a metadata type is given explicitly). Both default to an open record, so untyped usage is unchanged. Adds `@standard-schema/spec` as a dependency and bumps `futonic` to `0.1.1-canary.063ab81`, whose service constructor accepts the call-time endpoints-override type argument this relies on.
